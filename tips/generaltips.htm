Some tips to get happier with WINUAE

Set the CPU Idle slider to best values to save Money.

Modern CPU´s save much power when they have nothing to do,for example you write a text or  work with your workbench hear MP3 music.A AMD64 use in idle Mode 10 Watt.In full load Mode he use 83 Watt.A P4  Prescot with E0 Stepping have too the ability to save much power.A Athlon XP must disconnect the bus.Most BIOS have this not activate,because  old CPU´s have Bugs.But you can activate it and try it out,if system work stable

So use CPU idle save you money,give you lower temperature and if your fans are temperature controlled lower noise.

In the Tab CPU you can choose the optimal value for Power save.The CPU must be set to Fastest possible in A500 Mode winuae save always power

You must set the slider so far right,that the windows Task manager show for CPU load ca 10-15%.If you set it more right,you save more CPU load,but the clock run slower and winuae is slower then.But if you are on the road with a notebook it is maybe a good choose to give a little speed for longer accu life.The incorrect clock you can avoid with timehack program

Use hardfiles for more speed and only directory filesystem for stuff that you can share with windows (images music etc...)

Install OS3.9 when you have a running aiab (fastest way to get OS3.9 work)

If you dont have AIAB it is recommended to install it, it is much easier than installing from real Amiga emergency floppy.

  1. Boot from AIAB
  2. Insert the OS3.9 CD
  3. Start the OS3.9 Installer and choose "Make emergency floppy"
    The installer will create an emergency floppy with correct CD rom support and P96 support with correct rtg.library
  4. Boot from the emergency floppy
  5. You are ready to install OS3.9

Hint: If you get CD read errors you need reduce the max transfer rate in your CD-ROM driver settings.

How can i make custom screenresolutions for WinUAE

Powerstrip works OK now since Version 3.3 and nVidia drivers version 40 or later, so please make sure you are using the latest version of Powerstrip and nVidia drivers.

Create the modes:

  • Change into the menu displayprofile -> configure
  • Click advanced timing options -> custom resolutions -> user defined
  • Set desired values and reboot windows if necessary

It is also possible to create interlaced doublescan resolutions in this menu.

 

How can i install Os3.5 and 3.9 from emergency Floppy disk

It is only allowed to use OS3.5 or 3.9 on 1 Computer at the same time. Read the H&P license carefully. Maybe H&P placed a note in the license of newer OS3.9 distributions that restricts its use to AmigaHardware.

Here we go:

  1. Insert the emergency floppy in your old Amiga
  2. Use Transdisk to create an ADF and transfer it to your PC
  3. Boot your emulated Amiga with WinUAE and "insert" the ADF in a floppy drive
  4. Open the file devs/dosdrivers/emergency_CD on the disk with a texteditor
  5. Change the line device ="xxxxxx to device ="uaescsi.device"
  6. Change UnitNum if needed
  7. Change the text MaxTransfer = xxxxxxx to MaxTransfer =0xffff
    If you get read/write errors from CD use a value of 0x1fff instead
  8. Save the changed file to disk
  9. Boot from the emergency disk and make sure the OS3.5/3.9 CD is inserted

Harddrive performance

Use hardfiles as your main drive to avoid trouble, especially when using Windows 9x/ME for numerous reasons:

  • Amiga harddrives easily contain several 1000 files. This is too much for Windows 9x/ME because Windows needs to cache the directories, since FAT32 is slow. For example, if you try to delete the Windows directory from DOS without smartdrive, you will find out that this takes several hours.
  • The maximum number of files is limitet on FAT32.
  • Keep every hardfile smaller than 500 MB, because booting from a 1GB hardfile takes longer.
  • Also a backup from hardfiles is fast. 270MB takes 1 min with 1 Drive, 0.5 Min with two drives.

It is recommended to install the AmigaOS and programs on a hardfile, so the access and backup can be done quickly. To store pictures and other data, you may use a Windows directory for easier data exchange with the PC.
Transferring data from/to a hardfile can done by simply copying it, so you don't have to install all your Amiga stuff again.

UAE doesn't boot due to Picasso96 failure (Picasso96 is extremely slow)

Start UAE and immediately press both Mousebuttons when the main window is shown. In the Amiga Early Startup select "Boot without Startup-Sequence". Now remove or move the uaegfx file in devs:monitors. Start UAE, now the Workbench shows up. Just select a AmigaMode Save and copy uaegfx to devs/monitors again. If this should fail, reinstall Picasso96.

Don't forget to copy the rtg.library file, which can be found in the AmigaPrograms directory, to libs:picasso96.This  is important for full speed of winuae.

 

A chipset Mode that UAE does not display makes UAE fail

OS 3.1:

  • Boot without aStartup-Sequence (from Amiga Early Startup menu), type loadwb in the CLI, and press "Cancel" on the "ENV:" warnings
  • Start the screenmode selector
  • Select a standard PAL mode. Save it, reboot and select your preferred mode

OS 3.5/3.9:

  • Copy an old Screenmode prefs program from 3.1 to your HD, because the new one will not work this way.

 

UAE large boottime (memory usage)

As a rule of thumb, WinUAE needs 60+Z3+Chip+Fast+Picasso96 MB of system memory.
Since WinUAE 0.8.17r1, memory is allocated dynamically. This means you can select 512MB Z3 Mem and find no difference in boot time, but if you work with UAE for a longer time, it may get very slow. So it is better that you calculate your RAM needs with the above formula and make sure that it is lower than the pysical limit. Windows has to store the data on the harddisk if it runs out of RAM, which is a gigantic performance hit.
Also try to activate "Suspend to Disk" mode. UAE and Windows will be ready to use within 30 seconds.Dont forget to use the program  timehack to set the amigaclock correct

Tips for Tuning or random crashes

  • If you have an old HD (500MB), install a pure Windows for testing purposes. Does the program work ?
  • Overclock your PC by 10% by using the bus clock. If it works correctly your PC seems to be stable. If it works for 4 minutes, it seems to be stable, too. If it doesn't work, try downclock it. Less crashes?
  • A problem on non UDMA HD is that PIO4 or SCSI have no transfer error detection. So try to let your PC run on PIO3 (DMA still works) a month or so. The speed loss is not measurable in daily use, because Windows normally transfers small blocks. Less crashes?
  • You can install Sysmon by using the Windows installer. With this tooly you can see how many kB/sec are transferred and how much memory a program uses. So you see if the the program is slow is related to the memory. Normally the allocated memory should not exceed your physical memory by more than 30 MB. If so, you can increase speed by using more memory.
  • I found out that, large drives are better than drives with high RPM. A 9 GB 10000 RPM SCSI drive is slower than a 40GB 7200 RPM IDE drive, if you use the same partition size.
